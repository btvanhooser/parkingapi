Parking Lot API

GetLots

https://mysterious-atoll-40779.herokuapp.com/getLots [GET request]

Use this call to get a JSON object in return containing a JSON object as a value to the "results" key containing each lot tied to their IDs.



GetLot

https://mysterious-atoll-40779.herokuapp.com/getLot/{int: id} [GET request]

Use this request with any number 1 through 19 as the value of id. Each of these IDs are tied to an individual parking lot. Returned value is a JSON object with the key 'results' with its value being to a JSON object with the capacity info for the lot.



GetLotCapacity

https://mysterious-atoll-40779.herokuapp.com/getLotCapacity/{int: id} [GET request]

Use this request with any number 1 through 19 as the value of id. Each of these IDs are tied to an individual parking lot. Returned value is a JSON object with the key 'results' with its value being a string of either 'low', 'medium', or 'high', based on that specific lot's highest capacity value in the database.



GetAllLotCapacities

https://mysterious-atoll-40779.herokuapp.com/getAllLotsCapacities [GET request]

Use this request to get a JSON object back with the key 'results' and a JSON object with the name of each lot and their capactiy levels (either empty, mostlyEmpty, mostlyFull, or Full).



UpdateLot

https://mysterious-atoll-40779.herokuapp.com/updateLot [PUT request]

This request requires a payload containing a JSON object with the following keys and values: 'name': '{name of the lot}', 'capacityLevel': '{low, medium, or high}'. Returns a JSON object with the key 'results' and a value of either 'Lot successfully updated' if the lot exists, or 'No lot with that name exists' if a bad request was made. This request is intended to be a way for the user to update lot data in the database.



RefreshTable

https://mysterious-atoll-40779.herokuapp.com/refreshTable [GET request]

This request ensures that the database can be refreshed to account for new days. A seperate table is created that contains this API's last use (by date) and determines if today's date matches or not. If not, the database is rebuilt with every lot's data zeroed out. Otherwise, this request will not run the refresh process. A JSON object is returned with the key 'results' and the value of either 'Table successfully refreshed' if the current date is different, or 'Table is already refreshed' if this table is already refreshed for that day.
